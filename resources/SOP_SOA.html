Standard Operating Procedure (SOP)

SOA Reconciliation Module

Apeiron Bridge simplifies your Statement of Account (SOA) reconciliation. Follow this guide to match your SOA against multiple Reference files.

Phase 1: Launch & File Selection

1. Launch the Module
On the Welcome Screen, locate the "SOA Reconciliation" card and click the blue Launch button.
2. Add Excel/CSV Files
Click the green "Add Excel/CSV Files" button to browse, or simply drag and drop your files anywhere onto the list area. You can add as many reference files as you need.
3. Set the MAIN File (SOA) & Properties
  • Click the Radio Button on the far left of the row you want to act as your primary SOA (Main file).
  • Once selected, the Date and Amount selection boxes will appear for that file. Select the appropriate columns.
  • Tip: You can type a custom name (e.g., "Main SOA") into the text box next to the radio button to rename the file for the report.
4. Configure Reference Files & Ordering
  • For every other file in the list, select the column that matches the Master file using the Match Key / Join Column dropdown.
  • Tip: You can click and hold any file row to drag it up or down to change the sequence of the report columns.
5. Run Reconciliation (Recommended)
Click the purple "RUN RECONCILIATION" button to process immediately.
This skips the mapping screen and auto-maps all columns!
6. Optional: Advance Mapping
Click the dark "ADVANCE MAPPING" button only if you need to build normalized schemas or filter specific columns before running.

Phase 2: Advance Mapping (Optional)

Only used if you clicked "ADVANCE MAPPING".

1. Configure Base File (Left Panel)

  • Base File Columns: Review selections (Date/Amount are read-only here).

2. Configure Reference Files (Center Panel)

  • Select Reference File: Use dropdown to pick file.
  • Match Column: Verify the matching column.
  • Return Columns: Uncheck columns you don't want in the report.

3. Execution & Schema Configuration (Right Panel)

  • Fuzzy Match: Enable if IDs are messy (e.g., INV-100 vs 100).
  • Schema Configuration: Map Master names to Reference names to generate the 'Normalized Comparison' report.

Phase 3: Execution & Review

1. Run Reconciliation
Click the Run Reconciliation button (bottom right).
2. View Results
Check the tabs: Detailed View (Full Data) and Discrepancy Report (Variances).
4. Export Reports to Excel
Click "Export to Excel" to generate a multi-tab workbook containing the full breakdown.

Phase 4: Understanding The Insights Reports

The system generates 5 advanced AI-driven dashboards. Here is the logic behind the figures:

1. Executive Summary & KPIs

  • Match Rate %: Total number of fully matched invoices divided by total SOA invoices.
  • Discrepancy Value: The absolute sum of all financial differences (Deltas) between the Master and Reference files.
  • Health Score (0-100): Starts at 100 and is penalized up to 20 points based on the ratio of discrepancy value vs. the total SOA financial value.

2. Risk Scoring (Invoice Level)

Every mismatched invoice receives a Risk Score from 0 to 100 indicating audit priority:

  • Delta Magnitude (40 pts): The larger the financial difference compared to the maximum discrepancy, the higher the score.
  • Status Severity (25 pts): Missing records get max penalty (25 pts), while Partial Matches or minor mismatches get lower penalties (10-20 pts).
  • Missing References (15 pts): Penalizes invoices that are completely missing from 1 or more expected reference files.
  • Age (20 pts): Standard penalty logic applies to older unpaid invoices vs recent ones.

3. Aging Analysis

  • Bucketing: Invoices are grouped vertically into 30-day buckets (0-30, 31-60, 61-90, 121+).
  • Risk Levels: 0-30 days is marked 🟢 Safe. Moving past 60 days elevates to 🔴 High Risk. This helps Treasury teams prioritize collections.

4. Source Reliability

  • Coverage %: How many Master invoices actually exist in the reference file.
  • Accuracy %: Of the invoices that do exist, what percentage match the Master amount perfectly.
  • Grade (A+ to D): A+ requires >95% accuracy and >90% coverage. Grades drop dynamically based on missing data or bad amounts.

5. Pattern Detection

The AI Engine scans all financial deltas to find hidden behaviors:

  • Systematic Offsets: If >80% of a file's invoices are consistently under/over by a specific margin, it flags a "Systematic Gap".
  • Fixed Percentage Fees: Flags if discrepancies exactly match a fixed percentage (e.g., a hidden 2% platform fee).
  • Estimations: Flags sources where >50% of the financial amounts end in '00' (indicating estimated instead of actual billing).
